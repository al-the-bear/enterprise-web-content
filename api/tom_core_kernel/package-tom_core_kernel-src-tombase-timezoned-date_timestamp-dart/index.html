class TomTimezoned

class TomTimezoned extends DateTime

Base class for timezone-aware date/time values.

[TomTimezoned] extends [DateTime] to add timezone awareness. The internal DateTime is always stored in UTC, representing the actual moment in time. Accessors like [localYear], [localMonth], [localDay], [localHour], etc. return the local time values in the associated timezone.

Serialization Format

TomTimezoned values serialize to a string format that includes: - ISO 8601 datetime (always in UTC) - UTC offset at the time of serialization - Timezone name

Format: ISO8601_OFFSET_TIMEZONE Example: 2024-06-15T05:30:00.000Z_+09:00_Asia/Tokyo

Subclasses

Three specialized subclasses are provided: - [TomZonedDate] - For date-only values (time components are zero) - [TomZonedTime] - For time-only values (date is 1970-01-01) - [TomZonedDateTime] - For full datetime values

Each has its own serialization prefix for identification: - @D@ for dates - @T@ for times - @X@ for datetimes

Usage

// Create from local time with timezone
final tokyo = TomTimezone.fromName('Asia/Tokyo');
final tt = TomZonedDateTime(
  year: 2024, month: 6, day: 15, hour: 14, minute: 30,
  timezone: tokyo,
);
print(tt.localHour); // 14 (local time in Tokyo)
print(tt.hour);      // 5 (UTC time)

// Parse from serialized string
final parsed = TomTimezoned.parseFromString('2024-06-15T05:30:00.000Z_+09:00_Asia/Tokyo');

Creates a [TomTimezoned] representing the current time.

The current time is captured in UTC and associated with the provided timezone (defaults to UTC if not specified).

[timezone] The timezone to associate with this datetime.

Example:

Creates a [TomTimezoned] from an existing [DateTime].

The DateTime is converted to UTC before storing. The provided timezone is associated for display and conversion purposes.

Example:

class TomZonedDate

class TomZonedDate extends TomTimezoned

A timezone-aware date value (without time components).

[TomZonedDate] represents a calendar date associated with a specific timezone. All time components (hour, minute, second, etc.) are set to zero in the local timezone.

Serialization

Serializes with the @D@ prefix:

@D@2024-06-14T15:00:00.000Z_+09:00_Asia/Tokyo

(midnight June 15 Tokyo = 15:00 June 14 UTC)

Usage

// Create a date in a specific timezone
final tokyo = TomTimezone.fromName('Asia/Tokyo');
final date = TomZonedDate(year: 2024, month: 6, day: 15, timezone: tokyo);
print(date.localDay); // 15 (local day in Tokyo)

// Parse from string
final parsed = TomZonedDate.parseFromString('@D@2024-06-14T15:00:00.000Z_+09:00_Asia/Tokyo');
print(parsed.localDay); // 15 (local day in Tokyo)

Creates a [TomZonedDate] with the specified date components in the local timezone.

Time components are automatically set to midnight (00:00:00) in the local timezone. The values are then converted to UTC for internal storage.

[year] The year in local time (defaults to 1970). [month] The month 1-12 in local time (defaults to 1). [day] The day 1-31 in local time (defaults to 1). [timezone] The timezone for the local time values (defaults to UTC).

class TomZonedDateTime

class TomZonedDateTime extends TomTimezoned

A timezone-aware full datetime value.

[TomZonedDateTime] represents a complete date and time associated with a specific timezone. This is the most commonly used timezone-aware type.

Internal Representation

The datetime is stored internally as UTC. When you create a TomZonedDateTime with local time values, they are converted to UTC. Use [localHour], [localDay], etc. to get the local time values in the timezone.

Serialization

Serializes with the @X@ prefix:

@X@2024-06-15T05:30:00.000Z_+09:00_Asia/Tokyo

(14:30 Tokyo = 05:30 UTC)

Usage

// Create a datetime in a specific timezone
final tokyo = TomTimezone.fromName('Asia/Tokyo');
final dt = TomZonedDateTime(
  year: 2024, month: 6, day: 15,
  hour: 14, minute: 30,
  timezone: tokyo,
);
print(dt.localHour); // 14 (local time in Tokyo)
print(dt.hour);      // 5 (UTC time)

// Convert to another timezone
final ny = TomTimezone.fromName('America/New_York');
final dtNy = dt.convertToTimezone(ny);
print(dtNy.localHour); // 1 (local time in New York)

DST Handling

When creating datetimes in DST-aware timezones, the offset is determined at creation time based on the datetime value itself. 

final ny = TomTimezone.fromName('America/New_York');

// Summer (EDT: UTC-4)
final summer = TomZonedDateTime(
  year: 2024, month: 6, day: 15, hour: 12,
  timezone: ny,
);
print(summer.hour); // 16 (12 + 4 hours for EDT)

// Winter (EST: UTC-5)
final winter = TomZonedDateTime(
  year: 2024, month: 1, day: 15, hour: 12,
  timezone: ny,
);
print(winter.hour); // 17 (12 + 5 hours for EST)

class TomZonedTime

class TomZonedTime extends TomTimezoned

A timezone-aware time value (without date components).

[TomZonedTime] represents a time of day associated with a specific timezone. The date components are set to the Unix epoch (1970-01-01) as a placeholder.

Serialization

Serializes with the @T@ prefix:

@T@1970-01-01T05:30:00.000Z_+09:00_Asia/Tokyo

(14:30 Tokyo = 05:30 UTC)

Usage

// Create a time in a specific timezone
final tokyo = TomTimezone.fromName('Asia/Tokyo');
final time = TomZonedTime(hour: 14, minute: 30, timezone: tokyo);
print(time.localHour); // 14 (local time in Tokyo)
print(time.hour);      // 5 (UTC time)

// Parse from string
final parsed = TomZonedTime.parseFromString('@T@1970-01-01T05:30:00.000Z_+09:00_Asia/Tokyo');

DateTime inTimezoneOn(DateTime when)

Returns a [DateTime] representing this time on the given date.

This method applies the local time values (hour, minute, second, etc.) from this [TomZonedTime] to the date from [when], creating a full datetime in this timezone.

This is useful when you have a recurring time (e.g., daily standup at 9:00) and want to calculate the actual moment for a specific date, accounting for DST transitions.

[when] The date on which to apply this time. Only the date portion is used.

Returns a [DateTime] with this time on the given date in this timezone. The returned DateTime is actually a TZDateTime which provides timezone-aware local time values.

Example:

final standup = TomZonedTime(hour: 9, minute: 0, timezone: nyTz);

// Get the actual time for a specific date
final march10 = DateTime(2024, 3, 10);
final standupOnMarch10 = standup.inTimezoneOn(march10);
print(standupOnMarch10); // 2024-03-10 09:00:00.000-0400 (EDT after DST)
