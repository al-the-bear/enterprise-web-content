class TomClass

class TomClass extends TomObject<Map<String, TomObject<dynamic>>> implements TomObserver<TomObject<dynamic>>

A base class for complex observable objects with multiple [TomObject] members.

[TomClass] automatically registers all [TomObject] fields as observable members using reflection. When any member changes, the parent [TomClass] is notified and propagates the notification to its own observers.

Features

  • Automatic member registration: Fields are discovered via reflection
  • Cascading observation: Changes to any member notify the parent
  • JSON serialization: Built-in [toJson] and [fromJson] support
  • Batch updates: [setValues] for updating multiple fields at once
  • Muting: Temporarily suppress notifications during bulk updates

Example

@tomReflector
class Person extends TomClass {
  final name = TomString("");
  final age = TomInt(0);
  final email = TomNString(null);
}

final person = Person();
person.name | "John";
person.setValues({"age": 30, "email": "john@example.com"});
print(person.toJson());

class TomList

class TomList<E extends TomObject<dynamic>> extends TomObject<List<E>> implements TomObserver<TomObservable>, Iterable<dynamic>

An observable list that automatically observes all contained elements.

[TomList] wraps a [List] of [TomObject] elements and provides: - Automatic observation of all elements (changes in elements notify the list) - All standard [List] operations with observer management - Custom operators for convenient element manipulation

Methods

  • list.append(element) - Add an element (returns the list for chaining)
  • list[index] - Get element at index
  • list[index] = value - Set element at index
  • list + other - Concatenate two TomLists

Example

final items = TomList<TomString>();
items.append(TomString(\"first\"));
items.append(TomString(\"second\"));

items >> (obs) => print(\"List changed!\");
items[0] | \"updated\"; // Triggers observer

class TomListLifecycleListener

abstract class TomListLifecycleListener

Listener for structural reset events on a [TomList].

Register via [TomList.addLifecycleListener]. Implementors are notified around the [TomList.resetWith] batch-replacement operation so they can tear down state tied to the old elements before the list is cleared and re-create state for the new elements after they are installed.

This is the mechanism used by TomListForm in tom_flutter_ui to keep its sub-form instances in sync when TomClass.setValues() replaces the contents of a list member (e.g. when an outer form discards its changes and restores the bind-time snapshot).

Constructors

TomListLifecycleListener.new()

Methods

void onBeforeClear(TomList<TomObject<dynamic>> list)

Called immediately before all elements are removed from [list]. The list is still intact when this fires — implementors may read current elements for cleanup.

void onAfterReset(TomList<TomObject<dynamic>> list)

Called immediately after new elements have been installed in [list]. The list is fully populated when this fires — implementors may read the new contents to recreate state.

class TomObject

class TomObject<T> extends TomObservable

A generic observable wrapper that holds a value of type [T].

[TomObject] is the foundation of the observable type system. It wraps a value and provides: - Automatic observer notification when the value changes - Convenient operators for getting and setting values - Support for adding observer callbacks

Operators

  • ~obj - Get the value (prefix operator)
  • obj() - Get the value (call operator)
  • obj | newValue - Set the value and notify observers
  • obj >> callback - Add an observer callback function

Example

final name = TomString("John");

// Subscribe to changes
name >> (obs) => print("Name changed to: ${~obs}");

// Set new value (triggers observer)
name | "Jane";

// Get value
print(~name); // "Jane"
print(name()); // "Jane"

static TomObject<dynamic> _createFrom(dynamic v)

Creates a [TomObject] wrapper from a dynamic value.

Automatically determines the appropriate [TomObject] subtype based on the runtime type of [v]: - [String] → [TomString] (or timezone types if prefixed) - [DateTime] → [TomDateTime] - [int] → [TomInt] - [bool] → [TomBool] - [Map] → [TomMap] (recursively wraps values) - [List] → [TomList] (recursively wraps elements)

Returns [tomNull] if the type cannot be determined.
