Phase 5b — Cupertino Date / Time / Duration / Range concretes.

Five concretes that subclass the same Phase 3 abstract date/time fields used by the Material side, but render their inner picker as a CupertinoDatePicker / CupertinoTimerPicker inside a showCupertinoModalPopup sheet: [TomCupertinoFormDatePicker] → CupertinoDatePicker(date) [TomCupertinoFormTimePicker] → CupertinoDatePicker(time) [TomCupertinoFormDateTimePicker] → CupertinoDatePicker(dateAndTime) [TomCupertinoFormDurationPicker] → CupertinoTimerPicker * [TomCupertinoFormDateRangePicker] → two stacked date pickers

Surface: every concrete uses [TomGenericFieldDecorationMixin] so label / error / description / action icons render through the same anatomy as the Material counterparts. The on-tap tile renders via [cupertinoFieldDecoration] for visual parity with Phase 5a tiles.

function _cupertinoTriggerTile

Widget _cupertinoTriggerTile({required BuildContext context, required TomFieldStyle style, required IconData icon, required String label, required bool enabled, required void Function()?? onTap})

Renders a Cupertino-style trigger tile that displays [label] and a trailing chevron. Tapping invokes [onTap]. Uses the same surface decoration as the Phase 5a Cupertino picker tiles so date/time/range fields sit visually flush with string pickers.

function _showCupertinoPickerSheet

Future<void> _showCupertinoPickerSheet({required BuildContext context, required Widget picker, required void Function() onDone, double sheetHeight = 216})

Wraps [picker] in a modal-popup sheet with Cancel/Done buttons. Calls [onDone] only when the user taps Done; Cancel just closes.
