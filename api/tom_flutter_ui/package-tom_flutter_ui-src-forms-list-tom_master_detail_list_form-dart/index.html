class TomMasterDetailListForm

class TomMasterDetailListForm<TItem extends InvalidType, TForm extends TomForm<TItem>> extends TomListBase

Two-pane master-detail layout that consumes a [TomListForm] directly — no separate "detail form" instance is needed.

This is the no-sibling-form architecture of [TomInlineListForm] and [TomExpandableListForm] applied to the master-detail UX: when the user picks row N in the master pane, the detail builder is handed listForm[N] (the [TForm] the list itself owns). Because there is no second form bound to the same [TomClass], dirty state and value changes propagate cleanly through the standard child-container chain — the detail's isDirty rolls into the master list-form's isDirty, which rolls into the parent form's isDirty, and so on.

The widget is layout-agnostic: callers supply [masterBuilder] for the left pane and [detailBuilder] for the right pane. The master builder receives the current selection + a selectIndex callback that runs the unsaved-changes guard (when [guardUnsavedSelectionChange] is true) before rebuilding.

Example:

TomMasterDetailListForm<ShippingAddress, ShippingAddressForm>(
  listForm: form.shippingAddresses,
  masterPaneWidth: 220,
  masterBuilder: (ctx, list, selected, selectIndex) => AddressMasterPane(
    listForm: list, selectedIndex: selected, selectIndex: selectIndex,
  ),
  detailBuilder: (ctx, index, rowForm) => AddressDetailPane(form: rowForm),
)

Multiple [TomMasterDetailListForm] instances may share the same [TomListForm] — each maintains its own selection observable, so scaffold A may be looking at row 0 while scaffold B is looking at row 2. (See Demo H-A for the canonical two-scaffold-one-list demo.)

Resource resolution. When [tomId] is set, the following keys are consulted (each falls back to the constructor argument or a hard- coded default if no resource exists):

KeyTypeDefault
<basePath>.paddingEdgeInsets(no padding)
<basePath>.emptystringthe built-in italic hint
<basePath>.masterPaneWidthdouble280
<basePath>.gapdouble16
<basePath>.minMasterPaneWidthdouble120
<basePath>.maxMasterPaneWidthdouble(half of viewport)

Draggable splitter. Pass [draggableSplitter]: true to make the gap between the master and detail panes a horizontal-resize handle. The user can then drag the splitter to widen or shrink the master pane between [minMasterPaneWidth] and [maxMasterPaneWidth] (or, when [maxMasterPaneWidth] is null, half the available width). The starting width is [masterPaneWidth] (or its resource fallback).

Authorization. The whole widget passes through the standard [TomPolicyNodeBase] gate at <basePath> — none hides the entire scaffold (both panes), disabled greys it out and absorbs pointers, read applies the readonly visual treatment. Per-row authorization is not supported by design — every row of a list shares the list's auth state. Authorize the fields of the row sub-form via the field widgets themselves; the same field auth applies to every row.

Properties

final TomListForm<TItem, TForm> listForm

The list whose rows feed both panes.

final int? initialSelectedIndex

Optional initial selection. When non-null, the widget runs selectIndex(initialSelectedIndex) on the first frame so the detail pane comes up already showing that row.

The value is consulted only on the very first build; subsequent changes are ignored (selection becomes user-driven from then on).

final Widget Function(BuildContext, TomListForm<TItem, TForm>, int?, Future<void> Function(int?)) masterBuilder

Builds the master pane (typically a list of summaries plus an add/remove/reorder toolbar). The provided callback re-runs the unsaved-changes guard, applies the selection, and rebuilds the scaffold.

final Widget Function(BuildContext, int, TForm) detailBuilder

Builds the detail pane. The provided [rowForm] is the row's own sub-form (listForm[selectedIndex]) — bind any field widgets, Save/Discard buttons, error views directly to it.

When nothing is selected (or the index is out of range) this builder is NOT called — [emptyDetailBuilder] runs instead.

final Widget Function(BuildContext)?? emptyDetailBuilder

Optional widget shown when nothing is selected. Defaults to a centred italic hint (resolved from <basePath>.empty if set, otherwise the built-in fallback).

class _TomMasterDetailListFormImpl

class _TomMasterDetailListFormImpl<TItem extends InvalidType, TForm extends TomForm<TItem>> extends StatefulWidget

Internal stateful renderer that owns the selection observable and the [TomListForm] subscription. The public [TomMasterDetailListForm] resolves resources up-front and delegates to this widget so the auth / resource pipeline (which is StatelessWidget-shaped) and the live observer chain don't fight each other.
