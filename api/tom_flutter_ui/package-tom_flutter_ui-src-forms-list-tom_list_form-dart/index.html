TomListForm

package:tom_flutter_ui/src/forms/list/tom_list_form.dart

import 'package:tom_flutter_ui/src/forms/list/tom_list_form.dart';

class TomListForm

class TomListForm<TItem extends InvalidType, TForm extends TomForm<TItem>> implements TomFormEventSource, TomFormChildContainer

Container that manages an ordered collection of sub-forms — one per list element of the bound business object.

TomListForm is the form-side mirror of [TomList]. Where [TomList] observes the value channel of each contained [TomObject], TomListForm owns one [TomForm] instance per row and fans the form lifecycle — bind / save / discard / validate / dirty tracking — across every sub-form. It also publishes structural changes (add / remove / move) through the form event bus so [TomFormInspector] and logging UIs pick them up alongside per-field events.

Identity / resources follow the spec convention: every sub-form created by [formFactory] uses the same [listId] as its formId so resource lookups (order.items.quantity.label) apply uniformly to every row, rather than producing an indexed path per item.

Typical usage:

class OrderForm extends TomForm<Order> {
  OrderForm() : super(formId: 'order');

  late final orderNumber = TomFormStringInput('orderNumber', '', form: this);

  late final items = TomListForm<OrderItem, OrderItemForm>(
    listId: 'items',
    formFactory: (i) => OrderItemForm(),
  );

  @override
  void onBound() {
    items.bind((_binding! as Order).items);
  }
}

Methods

TForm add()

Append a new, blank sub-form. The parent's bound list (if any) gets a fresh [TItem] as well, instantiated by [createItem]. Subclasses must override [createItem] before calling [add] against a bound list.

TForm insert(int index)

Insert a blank sub-form at [index]. Same binding semantics as [add].

void removeAt(int index)

Remove the sub-form at [index] from this list and from the bound underlying list (if any).

bool remove(TForm form)

Convenience: remove a specific sub-form instance.

void move(int from, int to)

Move the sub-form from index [from] to index [to]. The bound list is reordered to match.

See [reorder] for the spec-canonical name; both forward to the same implementation and fire a [ListItemReorderedEvent].

void reorder(int from, int to)

Spec-canonical alias of [move] (§16: "reorder via drag handles drives .reorder"). Forwards to [move] so existing callers and events keep working unchanged.

void bind(InvalidType items)

Bind this list to a [TomList] of [TItem]. Tears down any sub-forms from a prior binding and re-instantiates one form per element.

Registers [this] as a [TomListLifecycleListener] on [items] so that structural resets driven by an outer TomClass.setValues() call (e.g. when an outer form discards its snapshot) automatically tear down the old sub-forms ([onBeforeClear]) and rebuild them for the new elements ([onAfterReset]).

void unbind() override

Detach from the bound list. Sub-forms are unbound and disposed.

void onBeforeClear(InvalidType list) override

Called by [TomList.resetWith] immediately before elements are removed. Disposes all current sub-forms so they are not left orphaned after the list contents are replaced (e.g. when an outer TomForm.discard() writes its snapshot back via binding.setValues()).

void onAfterReset(InvalidType list) override

Called by [TomList.resetWith] immediately after new elements are installed. Creates and binds fresh sub-forms for every element now in the list. Resets [_initialOrder] to the new contents so [isDirty] and [hasStructuralChanges] start from the correct baseline.

Future<List<FormValidationError>> validate() override

Validate every sub-form plus this list's own [validateList] hook. Returns the combined error list.

After computing errors a [FormValidatedEvent] is fired through the event bus (bubbling to the parent form when [parentForm] is set). This lets [TomErrorView] widgets subscribed to the parent form's bus pick up list-level errors — e.g. "at least one item required" — without needing a separate imperative update call.

Future<List<String>> validateList()

Optional list-level validation. Default returns an empty list.

Override to enforce things like minimum / maximum row count or duplicate checks across rows. Returned strings are treated as resource keys for cross-field error display (TomErrorView etc.).

Future<bool> save({bool? commitBaselines}) override

Save every sub-form. Returns true only when every sub-form saved successfully and [validateList] returned no errors.

[commitBaselines] mirrors [TomForm.save]'s policy parameter — when false, the per-row sub-form saves cascade with the same flag and _initialOrder is left untouched so an outer ancestor can still roll back structural changes via discard(). When null (default for direct user calls), the policy is auto-detected from the parent chain: a list-form whose parent form has its own ancestor in the tree is considered nested and skips the baseline update.

void discard() override

Discard edits in every sub-form AND restore the original list order / membership captured at bind / save time.

The restoration is driven by [TomList.resetWith] so that the [TomListLifecycleListener] hooks ([onBeforeClear] / [onAfterReset]) atomically dispose the current sub-forms and recreate fresh ones for the restored elements.

Order matters: sub-form field values are reset BEFORE [resetWith] so that [onAfterReset]'s bind() captures the correct (initial) TomClass state in the new sub-form's snapshot.

TItem createItem()

Override in subclasses to produce a fresh [TItem] when [add] / [insert] is called against a bound list. Default throws — only required if the demo / app uses [add] without pre-instantiating items in the bound list.
