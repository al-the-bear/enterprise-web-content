class TomExpandableListForm

class TomExpandableListForm<TItem extends InvalidType, TForm extends TomForm<TItem>> extends TomListBase

Renders a [TomListForm] as a vertical sequence of expandable rows — each row collapsed shows a builder-supplied summary, expanded shows a builder-supplied editor for the row's [TForm].

This is the right component when rows are individually heavy (many fields, validation panels, sub-tables) and the natural UX is "browse the list, drill into one row at a time". Each row sees the same [TForm] instance the [TomListForm] owns — there is no sibling form bound to the same [TomClass], so structural mutations and dirty state propagate cleanly through the standard child-container chain.

Expansion policy: [TomExpansionPolicy.singleOpen] (default) — opening a row collapses any other open row. [TomExpansionPolicy.multipleOpen] — multiple rows may be open concurrently.

Vetoing a state change: pass [onBeforeChange] to gate expansion toggles. Receives the row whose state would change and the desired new open state. Returning false cancels the toggle (the row stays in its current state). Useful for "validate before switching away" patterns.

External control: pass a [TomExpandableListFormController] to drive or observe expansion from outside (e.g. controller.collapseAll() after a successful save).

Resource resolution. When [tomId] is set, the following keys are consulted (each falls back to the constructor argument or a hard- coded default if no resource exists):

KeyTypeDefault
<basePath>.paddingEdgeInsets(no padding)
<basePath>.emptystring(no auto-empty widget)
<basePath>.policystring'singleOpen'

The resource value singleOpen / multipleOpen resolves to the matching [TomExpansionPolicy]; an explicit [policy] argument always wins over the resource.

Authorization. The whole widget passes through the standard [TomPolicyNodeBase] gate at <basePath> — none hides the entire list, disabled greys it out and absorbs pointers, read applies the readonly visual treatment. Per-row authorization is not supported by design — every row of a list shares the list's auth state. Authorize the fields of the row sub-form via the field widgets themselves; the same field auth applies to every row.

final Future<bool> Function(int, TForm, bool)?? onBeforeChange

Optional veto for an expansion state change. Receives the row that would change, its [TForm], and the desired new open state. Return false to cancel the toggle.

Common pattern (validate-before-switch):

onBeforeChange: (index, form, willOpen) async {
  if (willOpen) return true;          // collapsing — always allow
  final errors = await form.validate();
  if (errors.isEmpty) return true;
  showSnackbar(context, 'Fix errors first');
  return false;
}

When true, rows are wrapped in a [ReorderableListView] so the user can drag them into a new order via [TomReorderHandle] instances embedded inside [summaryBuilder]. class TomExpandableListFormController

class TomExpandableListFormController extends ChangeNotifier

Controls expansion state from outside the widget. The widget creates an internal controller when none is provided; pass your own when you need to react to or drive expansion (e.g. collapse all on save success, programmatically open the newly-added row).

All methods are async because an attached [TomExpandableListForm] may have an onBeforeChange veto that runs validation or shows confirmation dialogs.

void expand(int index, {TomExpansionPolicy policy = TomExpansionPolicy.singleOpen})

Programmatically expand [index] under [policy]. Does NOT consult the attached widget's onBeforeChange veto — use this for code- driven flows ("after save, open the new row") where bypassing the veto is the intended behaviour. For user-driven toggles, let the widget call its internal toggle so the veto runs.

void collapse(int index)

Programmatically collapse [index]. Does NOT consult any veto.

void collapseAll()

Programmatically collapse every row. Does NOT consult any veto. The canonical use is "after a successful save, collapse the open row(s) to return the user to the read-only summary view".

Internal: apply a state change without consulting any veto. Called by the widget after a veto check has passed (or when no veto is configured).

class _TomExpandableListFormImpl<TItem extends InvalidType, TForm extends TomForm<TItem>> extends StatefulWidget

Internal stateful renderer that owns the [TomListForm] subscription, the expansion controller, and the toggle/veto pipeline. The public [TomExpandableListForm] resolves resources up-front and delegates to this widget so the auth / resource pipeline (StatelessWidget-shaped) and the live observer / controller chain don't fight each other.

enum TomExpansionPolicy

enum TomExpansionPolicy

Expansion policy for [TomExpandableListForm].

Values

singleOpen

Opening a row collapses any other open row. The default — picks the spec's "no two dirty rows competing for attention" default for list editing.

multipleOpen

Multiple rows may be expanded at once. Useful when rows are independent and the user wants to compare or edit several in parallel.
