List<AclRow> _applyAuthHiding(List<AclRow> rows)

Removes components whose children are TomAuthorizable resolving to [TomAuthState.none] — provided either the container default ([AclContainer.defaultHideUnauthorized]) or the per-component [AclComponent.hideUnauthorized] override says so.

Elision rules (matching §26 of the ACL spec):

  • A hidden component's own gapBefore (the gap between it and the

previous visible component) is preserved and handed to the next visible component so the next visible component lands in the hidden component's slot. The next visible component's own gapBefore (the gap between the hidden component and itself — "the gap after the hidden component") is dropped. Chain of hidden components: only the first hidden component's gapBefore is carried forward; intermediate hidden components' gapBefores are "gaps after a previous hidden component" and get dropped. When every content component in a row is dropped, the whole row is dropped as if it didn't exist — its own gapBefore is carried to the next content row, which in turn has its own gapBefore dropped (it was the "gap after the elided row"). * Gap rows and components without a TomAuthorizable child are always preserved.

bool _shouldHide(AclComponent comp, bool containerDefault)

Whether [comp]'s child should be elided by the hide-unauthorized pass: the child must be [TomAuthorizable], the effective flag has to be true (per-component override or container default), and the resolved auth state must be [TomAuthState.none].
