class TomFieldStyle

class TomFieldStyle

The shared, fully-populated base of a [TomFormStyle].

Every per-widget-type style ([TomStringInputStyle], [TomCheckboxStyle], …) carries the same property shape but with every field nullable — acting as a partial override applied on top of this base.

Breakpoint-aware properties are expressed as two fields: the plain value and a matching (minWidth, value) overrides list. Resolution happens against the available width via [resolveForBp] / [pickPair].

Methods

T resolveForBp<T>(double width, T defaultValue, List<(double, T)> overrides)

Resolves a breakpointed (value, overrides) pair against [width].

Overrides are treated as sorted ascending by minWidth; the highest minWidth <= width wins. If none matches, [defaultValue] is returned.

T pickPair<T>(double width, T? overValue, List<(double, T)> overOverrides, T baseValue, List<(double, T)> baseOverrides)

Per-pair atomic pick: if [overValue] is non-null, resolve ([overValue], [overOverrides]) against [width]; otherwise resolve ([baseValue], [baseOverrides]).

The (value, overrides) pair is treated atomically — a sub-style cannot partially override only the list while inheriting the default.

class TomFormStyle

The umbrella style class — one instance per [TomFormStyleScope].

Holds a fully-populated [defaults] plus a nullable override per widget type. Consumers (fields / decoration mixins) read from both and pick per property via the per-pair atomic rule exposed by [TomFieldStyle.pickPair].

TomFieldStyle resolveFieldStyle(TomFieldPartialStyle? over, double width)

Resolves [defaults] + a per-widget [over] partial override down to a fully-populated [TomFieldStyle] at the given [width].

Applies the per-pair atomic rule from [TomFieldStyle.pickPair] to every breakpointable property and a simple over ?? base fallback to the rest. Returns the base unchanged when [over] is null.

TomFormStyle copyWith({TomFieldStyle? defaults, TomStringInputStyle? stringInputStyle, TomDropdownStyle? dropdownStyle, TomSliderStyle? sliderStyle, TomCheckboxStyle? checkboxStyle, TomSwitchStyle? switchStyle, TomSegmentedStyle? segmentedStyle, TomRadioGroupStyle? radioGroupStyle, TomDatePickerStyle? datePickerStyle, TomTimePickerStyle? timePickerStyle, TomFilePickerStyle? filePickerStyle, TomChipsStyle? chipsStyle, TomMultilineInputStyle? multilineInputStyle})
TomFormStyle withTheme(ThemeData theme)

Returns a derivative populated from [theme]'s color scheme and text theme. Only properties that still equal the preset's original defaults get replaced — user-set values are preserved.

This implementation compares each property against [TomFormStyle.material3]'s defaults to decide preservation: callers starting from any preset that shares those untouched defaults (material3, compact, bordered) get theme-aware colors without losing their own copyWith-applied overrides.
