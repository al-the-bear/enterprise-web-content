Phase 4 — Enum & Object selection fields.

Four abstract field types and their concretes: * [TomFormEnumField] / [TomFormObjectField] — single-value selection of an enum value or domain object from a [TomSelectableSource]. * [TomFormEnumListField] / [TomFormObjectListField] — multi-value variants (filter chips).

Enum and Object fields are structurally identical (both render a `TomSelectableSource<T>` of arbitrary `T`); they're separate types only so call-sites read clearly: enums are closed sets, objects are open sets typically driven by a domain repository. The runtime impl is shared via a generic mixin so we don't duplicate widget code.
