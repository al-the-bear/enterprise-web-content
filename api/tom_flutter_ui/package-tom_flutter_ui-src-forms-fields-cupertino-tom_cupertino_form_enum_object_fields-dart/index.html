Phase 5c — Cupertino Enum / Object selection concretes.

Three concretes that subclass the same Phase 4 abstract enum / object fields used by the Material side, but render through Cupertino primitives: [TomCupertinoFormEnumPicker] → CupertinoPicker modal sheet [TomCupertinoFormEnumSegmented] → CupertinoSlidingSegmentedControl * [TomCupertinoFormObjectPicker] → CupertinoPicker modal sheet

Enum and Object fields share the exact same widget shape — they only differ in the abstract base class, which exists so call-sites read clearly (closed enum sets vs. open domain objects). All three concretes use [TomGenericFieldDecorationMixin] so label / error / description / action-icon anatomy renders through the same column the Material counterparts use.

function _showCupertinoListPicker

Future<T?> _showCupertinoListPicker<T>({required BuildContext context, required List<SelectableItem<T>> items, required int initialIndex, double sheetHeight = 216})

Runs a Cupertino-style modal-sheet picker with a fixed Cancel/Done bar above a [CupertinoPicker]. Returns the selected value (or null on cancel). The picker's body is rendered inside a Cupertino popup so it matches the look of the Phase 5b date / time pickers.
