class TomAclBuilder

class TomAclBuilder<T extends TomScreenElementsProvider>

A delegating wrapper around [AclBuilder] that adds Tom UI concerns:

  • **Element provider** — typed access to screen elements via `(T p) => p.field`.
  • **Authorization** — container-level visibility check via [TomUIAuthorization].
  • **Resources** — container title resolved from [TomCtr] if not explicit.
  • **Identity scope** — wraps the result in [TomScope] for descendant resolution.

The wrapper delegates all layout operations to a standard [AclBuilder]. This keeps the layout engine independent of Tom-specific concerns.

Widget build(BuildContext context)

Builds the final widget tree.

Steps: 1. Check container-level authorization — if hidden, return [SizedBox.shrink]. 2. Resolve title from resources if not explicitly set. 3. Build the underlying [AclBuilder]. 4. Wrap in [TomScope] providing scope context to descendants.

ACL containers with IDs act as implicit group scopes — any widget inside can use this container's ID as part of their canonical path.

The returned widget must be placed in a [BuildContext] that has a [TomThemeScope] ancestor for readonly styling (falls back to default).
