AclBuilder newRow({double dist = 0, double? height, List<(double, double?)> heightBpOverrides = const []})

Starts a new content row.

Optional [height] fixes the row's rendered height (null falls back to the builder's default set via [setAclDefaultRowHeight]). [heightBpOverrides] allows per-breakpoint overrides following the standard `(minWidth, value)` resolution rule.

AclBuilder gapRow({double weight = 1, AclSeparator? separator, double minSize = 0})

Inserts an elastic vertical gap row; **implicitly starts a new content row**.

In scrollable-Y mode, the gap row collapses to zero height.

When the layout also contains [expandY] components, gap rows yield all extra vertical space to those rows. In that case: * A [separator] gapRow renders at [minSize] height (minimum 9 px). * A pure spacing gapRow renders at [minSize] height, or collapses to nothing if [minSize] is 0 (the default).

AclBuilder setAclExpandXDefault(bool enabled)

Sets the default expand-X behaviour for newly appended components.

When [enabled] is `true`, [AclFlags.expandX] is added to the default flags so every subsequent [append] call gets expandX automatically. When `false`, the flag is removed from the defaults.

Individual components can still override via [clearAttr]/[setAttr].

AclBuilder setAclDefaultAppendGap(double gap, [List<(double, double)> gapBpOverrides = const []])

Sets the default horizontal gap inserted before every appended component of every row, including the first component.

Applies only when the individual [append] call does not specify a non-zero `dist`. Pass [gap] = 0 to disable (the built-in default is 8).

[gapBpOverrides] is a list of `(minWidth, gap)` pairs; at render time the pair with the highest `minWidth <= availableWidth` wins.

AclBuilder setAclDefaultRowHeight(double? height, [List<(double, double?)> heightBpOverrides = const []])

Sets the builder-wide default row height (null = intrinsic).

Applied to every row that does not set its own height via [setRowHeight] or [newRow]'s `height:` parameter. [heightBpOverrides] is a list of `(minWidth, height?)` pairs.

AclBuilder setRowHeight(double? height, [List<(double, double?)> heightBpOverrides = const []])

Sets the height on the current (most-recently-started) row.

[heightBpOverrides] follows the same `(minWidth, height?)` shape as [setAclDefaultRowHeight]. Passing `null` height with an empty overrides list clears the row-level override (the builder default applies).

AclBuilder setAclDefaultRowGap(int pixels)

Sets the default vertical gap (in pixels) inserted before each new content row started with [newRow].

Only applies when the individual [newRow] call does not specify a non-zero `dist`. Pass 0 to disable (the default).

AclBuilder setAclDefaultHideUnauthorized(bool value)

Sets the builder-wide default for the hide-when-unauthorized behaviour. When `true`, every appended component whose child implements [TomAuthorizable] and currently evaluates to [TomAuthState.none] is **dropped from the row** at build time; the `gapBefore` of the component that follows it is also dropped so the next visible component takes the hidden component's slot. When a row is left with no components it is dropped in turn, along with the next row's `gapBefore`.

The default is `false` — components still render as whatever their own self-shrink-on-`none` path produces (typically a `SizedBox.shrink()` placeholder inside a preserved gap).

Override per-component with [setHideUnauthorized].

AclBuilder setHideUnauthorized(bool? value)

Overrides the container's hide-when-unauthorized default for the most recently appended component. Pass `null` to clear the override and fall back to the container default.

See [setAclDefaultHideUnauthorized] for the full semantics.

AclBuilder setAclDefaultRowVerticalAlignment(AclRowCrossAlignment alignment)

Builder-wide default for the **vertical** cross-axis placement of every row. Individual rows override via [setRowVerticalAlignment]; individual components override via [setVerticalAlignment].

The factory default is [AclRowCrossAlignment.top] — chosen so that fields whose height grows (e.g. when an error message appears below the input) do not drag their neighbours off the baseline.

AclBuilder setAclDefaultRowHorizontalAlignment(RowAlignment alignment)

Builder-wide default for the **horizontal** main-axis placement of every row. Individual rows override via [setRowHorizontalAlignment] or the Left/Center/Right shortcuts.

AclBuilder setAclDefaultUnknownMinimumSize(double width, [double height = 0])

Sets a default minimum size for components that do not have an explicit minimum size ([setMSize] / [setMWidthInChars]).

Pass 0 for [width] or [height] to leave that dimension untouched (i.e. no default minimum is applied on that axis).

**Note:** setting a minimum that is smaller than the preferred size has no visible effect — the preferred size already acts as the floor. Setting a minimum that is *larger* than the preferred size in combination with [expandX] components creates a "grow up to preferred, then distribute remainder to expandX" behaviour (see §6).

AclBuilder setAclDefaultUnknownPreferredSize(double width, [double height = 0])

Sets a default preferred size for components that do not have an explicit preferred size ([setPSize] / [setPWidthInChars]).

Replaces the built-in default of 150 × 0. Pass 0 for [width] to keep the built-in default width (150); pass 0 for [height] to keep the default height of 0 (unconstrained).

static double _defaultPreferredWidth

Default preferred width for components without an explicit preferred size and no custom [_defaultUnknownPreferredSize]. Large enough to prevent buttons/small widgets from collapsing next to elastic gaps, but still participates proportionally in the grow/shrink distribution alongside explicitly sized components.
