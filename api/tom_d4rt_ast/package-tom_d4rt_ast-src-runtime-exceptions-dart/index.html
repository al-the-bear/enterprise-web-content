class D4UnwrapException

class D4UnwrapException extends D4rtException

Exception thrown by [D4.unwrapAs] when an interpreter result cannot be converted to the requested native type [T].

Carries enough context (the requested type description, the actual runtime type, and an optional source description) to make the failure actionable for embedders that translate interpreter results to native Dart values.

Example:

try {
  final widget = D4.unwrapAs<Widget>(result, visitor: visitor);
} on D4UnwrapException catch (e) {
  print(e); // -> Expected Widget but got String (from raw value)
}

Properties

String description of the expected type (typically T.toString()).

final String actualType

String description of the value's actual runtime type.

final String? source

Optional human-readable description of where the value came from (e.g. BridgedInstance<Widget> or InterpretedInstance(MyClass)).

class D4rtDiag

class D4rtDiag

Lightweight, always-on interpreter instrumentation counters.

These are plain integer increments on the hottest interpreter paths — no allocation, no stack capture — so they are safe to leave compiled in. They exist to diagnose pathological per-frame cost in embeddings (e.g. the particle_field Flutter sample freeze): a harness reads [callCount] / [maxDepth] before and after a unit of work to see whether a spike is driven by call volume or interpreter recursion depth (the latter is what makes exception-based return unwinding expensive — see the perf analysis).

Constructors

D4rtDiag.new()

Static

static int callCount

Monotonic count of interpreted function-call entries (_callImpl).

static int depth

Current synchronous interpreter call-nesting depth.

static int maxDepth

High-water mark of [depth] since the last [reset].

static int envAllocs

Per-window allocation counters for the prime per-rebuild garbage suspects. The particle_field freeze is a stop-the-world major GC of ~2.2GB old-gen garbage accumulated at ~22MB/frame; these counters localize which interpreter allocation dominates a single interpreted rebuild.

static int closureAllocs

Count of interpreted-closure objects materialized in the window.

static int instanceAllocs

Count of InterpretedInstance objects constructed in the window.

static int bridgedAllocs

Count of BridgedInstance objects constructed in the window.

static void reset()

Resets the windowed counters ([callCount], [maxDepth], and the allocation counters). [depth] is the live nesting level and is intentionally left untouched.

static void enterCall()

Records entry into an interpreted call frame.

static void exitCall()

Records exit from an interpreted call frame.

class D4rtException

abstract class D4rtException implements Exception

Base class for all D4rt-specific exceptions that should be tracked.

All exceptions are automatically registered with [ErrorReporter] on creation. If an exception is caught and handled gracefully, call [revoke()] to prevent it from causing test failures.

Constructors

D4rtException.new(String message)

Creates a [D4rtException] and registers it with the [ErrorReporter].

Properties

final String message

The error message.

StackTrace? trackedStackTrace

Stack trace captured by [ErrorReporter] when tracking is enabled.

Stored on the instance (rather than in a side map) so revoking an error is a single O(1) set removal with no parallel map bookkeeping. Null when tracking was off at construction time or after the error was revoked.

Methods

bool revoke()

Revokes this error from the ErrorReporter.

Call this when the exception has been caught and handled gracefully, and should not count as a test failure.

Returns true if the error was successfully revoked, false if it was not found in the reporter (possibly already revoked).

Example:

try {
  // Some code that might fail
  d4rt.eval('badCode');
} catch (e) {
  if (e is SourceCodeException) {
    // Handle the error gracefully
    print('Handled parse error');
    e.revoke(); // Don't count this as a test failure
  }
}

class ErrorReporter

class ErrorReporter

Static error reporter that tracks all created [D4rtException]s.

This is used primarily in test modes to detect errors that occurred during script execution.

Errors can be revoked if they are caught and handled gracefully, preventing them from causing test failures.

Constructors

ErrorReporter.new()

Static

static Set<D4rtException> _errors

Identity set of tracked errors.

An identity [Set] makes [revokeError] an O(1) remove instead of the O(n) linear scan a List.remove performed — that scan was the dominant interpreter CPU cost (see the perf analysis). Exceptions are unique objects, so identity membership matches the prior list semantics exactly. Insertion order is preserved (identity sets are linked), so [report] ordering is unchanged.

static bool _trackingEnabled

Tracking is off by default. It is an opt-in debugging/test aid: capturing StackTrace.current plus bookkeeping on every exception during normal execution is pure overhead. Test harnesses and the REPL call [enableTracking] when they need the report.

static bool isTrackingEnabled
static List<D4rtException> errors
static bool hasErrors
static String report
static String reportShort
static void reportError(D4rtException error, [StackTrace? stackTrace])

Reports an error to the reporter (only if tracking is enabled).

This is called automatically by the [D4rtException] constructor. The stack trace is stored on the exception instance itself (not in a side map), so no parallel Map.remove is needed on revoke.

static void disableTracking()

Temporarily disables error tracking.

This is useful during speculative operations (like trying to parse as expression before falling back to statement) where errors are expected and should not be tracked.

static void enableTracking()

Enables error tracking. Off by default; opt in from tests/REPL.

static bool revokeError(D4rtException error)

Revokes (removes) an error from the reporter.

This should be called when an exception has been caught and handled gracefully, and should not count as a test failure.

Returns true if the error was found and removed, false otherwise.

static void revokeAll()

Revokes all currently tracked errors.

This is useful when a batch of errors occurred during a failed attempt (like trying to parse as expression before falling back to statement), and all those errors should be discarded.

static void clear()

Clears all reported errors.

class RuntimeD4rtException

class RuntimeD4rtException extends D4rtException

Custom exception for runtime errors during interpretation.

This exception is thrown when the interpreter encounters an error during code execution, such as accessing undefined variables, calling non-existent methods, or type mismatches.

Constructors

RuntimeD4rtException.new(String message, {Object? originalException})

Creates a new runtime error with the given message. When [originalException] is supplied, the wrapped native exception stays catchable by type.

Properties

final Object? originalException

When this runtime error wraps a native exception thrown from a bridged adapter (method, static method, getter, setter, operator, constructor), the original exception object is preserved here. This lets a typed on NativeType / bare catch clause in interpreted code dispatch against the real exception type rather than this RuntimeError wrapper. (OPEN B.5 — U13/U24.)
