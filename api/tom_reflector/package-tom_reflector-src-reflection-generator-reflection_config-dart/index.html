class MarkerAnnotationConfig

class MarkerAnnotationConfig

Configuration for marker annotation scanning.

This allows discovering all types marked with specific annotations, similar to how dependency injection frameworks work.

Constructors

const MarkerAnnotationConfig.new({bool enabled = false, List<String> markerAnnotations = const [], List<String> scanPackages = const [], bool followAnnotationChains = true})

Properties

final bool enabled

Whether marker annotation scanning is enabled.

final List<String> markerAnnotations

Annotation names to treat as markers (e.g., 'tomReflection').

When one of these annotations is discovered (either directly or transitively), scan the codebase for all types using it.

final List<String> scanPackages

Package patterns to scan for marked types (glob patterns).

If empty, only scans packages already in scope.

final bool followAnnotationChains

Whether to follow annotation chains.

If true: @tomReflectionInfo uses tomReflection, so finding tomReflectionInfo also triggers scanning for tomReflection markers.

static MarkerAnnotationConfig fromMap(Map<String, dynamic> map)

class ReflectionConfig

class ReflectionConfig

Configuration for reflection generation.

This class represents the complete configuration for the reflection generator, parsed from tom_analyzer.yaml or build.yaml.

Constructors

const ReflectionConfig.new({List<String> entryPoints = const [], String? output, ReflectionDefaults defaults = const ReflectionDefaults(), List<ReflectionFilter> filters = const [], DependencyConfig dependencyConfig = const DependencyConfig(), CoverageConfig coverageConfig = const CoverageConfig(), SourceExtractionConfig sourceExtractionConfig = const SourceExtractionConfig(), bool includePrivate = false, Map<String, dynamic> raw = const {}})

Properties

final List<String> entryPoints

Entry point files for analysis.

These are the starting points for reachability analysis. All types reachable from these files are candidates for reflection.

final String? output

Output file path (without .r.dart extension).

The generator will add .r.dart automatically. If null, defaults to the first entry point path with .dart replaced.

final ReflectionDefaults defaults

Default settings applied before filters.

final List<ReflectionFilter> filters

Filters to apply in order.

final DependencyConfig dependencyConfig

Configuration for transitive dependency inclusion.

final CoverageConfig coverageConfig

Configuration for coverage (what invokers to generate).

final SourceExtractionConfig sourceExtractionConfig

Configuration for source code extraction (optional, memory-intensive).

final bool includePrivate

Whether to include private members (default: false).

final Map<String, dynamic> raw

Raw configuration map (for extensions).

bool hasMultipleEntryPoints
bool shouldCombineOutput

Methods

class TypeAnnotationConfig

class TypeAnnotationConfig

Type annotation inclusion configuration.

Constructors

const TypeAnnotationConfig.new({bool enabled = true, bool transitive = false, bool external = true, bool includeArgumentTypes = true, bool scanMarkedTypes = false})

Properties

final bool enabled

Whether to include type annotations.

final bool transitive

Whether to follow annotations transitively.

If true, when an annotation type is discovered, also discover annotations used on that annotation type (meta-annotations).

final bool external

Whether to include external annotation types.

final bool includeArgumentTypes

Whether to include types referenced in annotation arguments.

For example: @MyAnnotation(SomeClass) would include SomeClass.

final bool scanMarkedTypes

Whether to scan for all types marked with discovered annotations.

If true, when a marker annotation like @tomReflection is discovered, the analyzer will scan the codebase for all types using that annotation.
